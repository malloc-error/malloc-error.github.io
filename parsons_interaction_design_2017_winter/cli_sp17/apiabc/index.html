Summary

Look into one or more web APIs you might be interested in experimenting with for a project. Make a simple report on each API including: its name, the company that sponsors it, basic overview of the data/content that you can get from it, how you could use it in a project, link to the documentation/instructions, the URL to access it from your code, how to get an access key.

Additional Details

Here is an example for Google Maps API (formatting/style are up to you, but must be clear and easily legible):
Name: Google Maps API

Parent Company: Google (not all of them will be this obvious)

Data/Content: Map tiles, street images, pin locations of businesses/addresses

Possible Uses: I could make a Kombucha-themed web page with a map of my favorite places to buy the drink and/or ingredients to make it

Documentation: https://developers.google.com/maps/web

URL to access the API: https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY

Getting a key: https://developers.google.com/maps/documentation/javascript/
Click "Get a Key" — Free and paid plans available

Hint

You don't have to actually code anything in this stage, so spend your time exploring. Don't do a write-up on the Google/YouTube API. Even searching for "fun web apis" or "popular apis" will yield many results to look into.

If you have a specific topic you'd like to try finding an API for, search something like "api for recipes" or "weather web api," etc.

You won't have to use this API for the next assignment, but you can. So be thoughtful in your explorations. If you can find an interesting API that is simple enough to work with, you won't have to search again from scratch for the next assignment.
