Summary

Create a web experience that responds to the size of the browser viewport. While the content may be simple and technologically basic, it should be interesting and intentionally designed.

Additional Details

Your page should have at least one media query breakpoint (e.g. mobile vs desktop) based on the size of the browser window, as demonstrated in class.

You are encouraged to use percentages to control width, but if it becomes confusing, you may use media query breakpoints with multiple hardcoded (pixel) widths.

Your design should include: an element (img, h1, etc.) that changes size, an element that hides/shows, and an element that changes position, all based on window width.

This project should build off your previous CSS knowledge as well as your design skills to date. It can be simple, but must be well designed, with thoughtful choices of color and typography. A page that only meets technical requirements but has no consideration for graphic design may be returned for a redo.

Hints and Ideas

Possible project ideas include: a short informational article, a shop page, instructions, directions to a place, a recipe.

Try to find ways to make the desktop and mobile versions complement or interact with each other. Think about how someone might use one version differently from the other.
